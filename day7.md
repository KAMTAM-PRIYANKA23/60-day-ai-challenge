# My Personalized AI Workflow

## Learning

1. Learn concept using Claude
2. Request simple explanation
3. Study examples
4. Implement independently
5. Ask Claude for review

---

## Coding

1. Understand project flow
2. Break project into modules
3. Implement one module
4. Test independently
5. Use AI only when stuck

---

## DSA

1. Understand concept visually
2. Dry-run examples
3. Solve easy problems
4. Write code from memory
5. Review mistakes

---

## Projects

1. Create architecture
2. Divide into modules
3. Build module-by-module
4. Document progress
5. Push regularly to GitHub

# Key Learnings

* Consistency beats intensity.
* Projects should be learning tools, not just submissions.
* AI should accelerate understanding, not replace it.
* GitHub documentation is as important as coding.
* Strong fundamentals create long-term career advantages.
